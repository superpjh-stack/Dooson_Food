# [UI2] 생산·LOT 화면 설계서

**작성자**: Designer2 (Frontend Architect — Production/LOT UX)
**작성일**: 2026-05-04
**버전**: v1.0
**프로젝트**: 두손푸드 AI-MES
**단계**: PDCA Design
**관련 Plan**: PM1 (생산·LOT 트레이서빌리티 기획서)

---

## 1. 설계 범위

- 생산 계획 캘린더
- 작업 지시서 발행 및 모바일 작업 지시 (작업자 태블릿)
- LOT 검색/추적 (정·역방향) + LOT 트리 시각화
- BOM 트리 등록/수정 + BOM 버전 비교
- 공정 실적 입력 (NFC/바코드)

## 2. 디자인 원칙

- **현장 태블릿 우선**: 10인치 태블릿(1280x800)이 가장 빈번한 디바이스 → 큰 터치 타겟(최소 48px), 한 화면 내 핵심 동작 완결
- **데이터 정합성 우선**: 생산/LOT 데이터는 변경 시 항상 검증 후 저장 (Optimistic UI 신중 적용)
- **시각적 위계**: LOT 트리·BOM 트리는 Indent + Color로 깊이 표현
- **색상**: UI1 토큰 일관 사용 (`--color-primary` 두손푸드 그린 외)

### 5.2 핵심 UX
- **큰 터치 타겟**: 모든 버튼 최소 48px (장갑 끼고도 조작)
- **수량 입력**: NumPad UI + ±버튼 (실수 방지)
- **자동 LOT 인식**: 바코드/QR/NFC 스캔으로 즉시 입력
- **오프라인 모드**: 통신 두절 시 로컬 캐싱 + 화면 상단 노란 띠 ("오프라인 - 동기화 대기 3건")
- **단계 안내**: 현재 공정/다음 공정 시각적 강조

## 11. 폼 설계 원칙

- **인라인 검증**: 입력 즉시 검증 (debounce 300ms)
- **에러 메시지**: 필드 하단에 빨간 텍스트 + 아이콘
- **자동 저장**: 30초마다 임시 저장 (작성 중인 폼)
- **확인 모달**: 파괴적 작업(삭제, 회수)은 항상 2단계 확인
- **단축키 안내**: Ctrl+S 저장, Esc 취소 (현장 키오스크 제외)
