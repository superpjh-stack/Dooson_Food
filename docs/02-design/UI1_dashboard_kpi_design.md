## 1. 설계 범위

이 문서는 두손푸드 AI-MES의 다음 화면을 설계함:
- **메인 KPI 대시보드** (공장장/경영진)
- **부서별 KPI 상세** (부서장)
- **AI Agent 챗봇 통합 패널**
- **실시간 알림 센터**
- **경영 보고서 자동 생성**

## 2. 디자인 원칙

### 2.1 톤 & 매너
- **Industrial Modern**: 식품 제조 현장의 신뢰감 + 디지털 시대의 명료함
- **High-density Information**: 화면 한 장에 핵심 KPI 한눈에 (콕핏 메타포)
- **WCAG 2.1 AA**: 색약자 고려, Color + Icon + Text 3중 표현

### 2.2 컬러 팔레트
| 토큰 | HEX | 용도 |
|------|-----|------|
| `--color-primary` | `#1B5E20` | 두손푸드 그린 (식품/자연 메타포) |
| `--color-secondary` | `#0F4C81` | 공업 블루 (제조 신뢰감) |
| `--color-success` | `#2E7D32` | 정상 상태 |
| `--color-warning` | `#F57F17` | 경고 (CCP 임계 근접) |
| `--color-danger` | `#C62828` | 위험 (CCP 이탈, 설비 고장) |
| `--color-info` | `#0277BD` | 정보 알림 |
| `--bg-base` | `#F5F7FA` | 배경 |
| `--bg-surface` | `#FFFFFF` | 카드 배경 |
| `--text-primary` | `#1A1A1A` | 본문 |
| `--text-muted` | `#5F6368` | 보조 |

### 2.3 타이포그래피
- **본문**: Pretendard (한글), Inter (영문/숫자)
- **숫자**: Tabular figures (KPI 정렬 위해)
- **위계**: H1 32px / H2 24px / H3 18px / Body 14px / Caption 12px

### 2.4 그리드 시스템
- 12-column grid, gutter 16px
- Card-based layout, Border-radius 8px
- Spacing tokens: 4 / 8 / 12 / 16 / 24 / 32 / 48px

|  - "다른 살균기와 비교"                |
+----------------------------------------+
| [ 입력... 자연어로 물어보세요   ] [↑] |
| [🎤 음성]                              |
+----------------------------------------+
```

### 5.2 핵심 UX 패턴

1. **항상 출처 표시**: AI 답변 하단에 데이터/문서/모델 출처 명시 (사용자 신뢰 확보)
2. **팔로업 제안 (Suggested Replies)**: AI가 다음 질의 옵션 3개 추천
3. **Action Chips**: "상세 보기", "차트", "PDF 다운로드" 등 답변 기반 액션 버튼
4. **모드 전환**: Floating ↔ Full-screen ↔ Sidebar (Drawer)
5. **세션 이력**: 좌측 Drawer에 이전 대화 세션 목록
6. **응답 스트리밍**: SSE/WebSocket으로 토큰 단위 실시간 표시
7. **Hallucination 방지 UI**: AI가 확신 없을 때 "확인 필요" 라벨, Confidence 표시

+----------------------------------------+
| ❗ 긴급                                |

|       LSTM 예측: 12h 이내 고장 가능    |
|       [정비 요청] [무시] [상세]        |
| ────                                   |
| ⚠ 경고                                 |
| 14:21 [품질] CCP-3 임계 근접          |

| ────                                   |
| ℹ️ 정보                                |
| 14:15 [생산] WO-2026-1245 완료         |

+----------------------------------------+
```

### 6.2 알림 우선순위 + 채널 매트릭스
| 우선순위 | 색상/아이콘 | 채널 |
|---------|-----------|------|
| Critical | 빨강 ❗ | 푸시 + SMS + 이메일 + 슬랙 |
| Warning | 주황 ⚠ | 푸시 + 슬랙 |
| Info | 파랑 ℹ️ | 푸시만 |

### 6.3 인터랙션 흐름
1. 신규 알림 도착 → Header `[🔔]` 배지 카운트 증가 + Subtle Toast (3초)
2. Critical 알림 → Modal Confirm (사용자 인지 확인 필요)
3. 알림 클릭 → 관련 화면으로 컨텍스트 전환

## 9. 반응형 (Responsive)

| 디바이스 | 폭 | 레이아웃 |
|---------|-----|---------|
| Desktop | ≥ 1280px | 12-col 풀 |
| Laptop | 1024 ~ 1279 | 12-col 압축 |
| Tablet | 768 ~ 1023 | 6-col, KPI 카드 2x2 |
| Mobile | < 768 | 1-col, 핵심 KPI만 노출 |

- **현장 태블릿** (10인치, 1280x800)을 1순위로 최적화
- **공장장 데스크톱** (24인치, 1920x1080)을 2순위로 최적화

## 10. 접근성 (Accessibility)

- **키보드 내비게이션**: 모든 액션 Tab/Enter 가능
- **스크린리더**: ARIA Label 필수 (특히 차트는 데이터 표 대체 제공)
- **색약자**: 색상에만 의존 금지, Icon+Text 함께 표기
- **명도 대비**: WCAG AA 4.5:1 이상
