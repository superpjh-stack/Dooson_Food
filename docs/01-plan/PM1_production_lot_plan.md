## 1. 개요 (Overview)

### 1.1 배경
두손푸드는 식품 제조 전문기업으로, 현재 PLC 연동·LOT 기반 생산 추적·공정 모니터링이 모두 **수기/수동 방식**으로 진행되고 있어 다음과 같은 문제가 발생함:

- 공정 데이터의 실시간 가시성 부재
- LOT 추적 시 수기 기록 누락 및 오기 발생 (월 평균 3~5건)
- BOM 변경 시 현장 반영까지 평균 2일 소요
- 불량 발생 시 원인 LOT 역추적 시간 평균 4시간 이상

### 1.2 목표 (Objective)
**MES + IoT 자동화를 통한 LOT 단위 실시간 생산 추적·BOM 기반 자동 생산 지시·공정별 실적 자동 수집 체계 구축**

### 1.3 성공 지표 (KPI)
| 지표 | 현재(AS-IS) | 목표(TO-BE) | 달성 기한 |
|------|------------|-------------|----------|
| LOT 추적 자동화율 | 0% | 95% | 2026-12 |
| 공정 실적 자동 수집률 | 0% | 90% | 2026-12 |
| LOT 역추적 소요 시간 | 4시간 | 5분 이하 | 2027-03 |
| BOM 현장 반영 시간 | 2일 | 즉시(실시간) | 2026-12 |
| 생산 지시 발행 시간 | 30분/건 | 1분/건 | 2026-10 |

## 2. 사용자 페르소나 (Personas)

### Persona A: 생산팀장 (이상철, 45세)
- **역할**: 일일 생산 계획 수립, 작업 지시, 현장 관리
- **Pain Point**: 작업자별 진척도 파악이 늦어 라인 효율 저하
- **Needs**: 실시간 라인별 생산 진척률 대시보드, 모바일 작업 지시

### Persona B: 현장 작업자 (박지연, 32세)
- **역할**: 제조 라인 작업, LOT 입력, 실적 기록
- **Pain Point**: 종이 작업지시서·수기 LOT 기록의 번거로움
- **Needs**: 태블릿/바코드 기반 LOT 자동 입력, 작업지시 푸시 알림

### Persona C: 자재팀장 (김혜영, 38세)
- **역할**: 원자재 입고·BOM 관리·재고 관리
- **Pain Point**: BOM 변경 시 현장 동기화 지연, 자재 소요량 예측 부정확
- **Needs**: BOM 버전 관리, 자재 자동 차감, 부족 알림

### Persona D: 품질팀 LOT 추적 담당 (정훈, 41세)
- **역할**: 불량 발생 시 LOT 역추적, 회수(Recall) 관리
- **Pain Point**: 수기 기록 누락 시 추적 불가, 원료~완제품 연결 어려움
- **Needs**: 풀 트레이서빌리티(원료→공정→완제품→고객) 자동 추적

## 3. 사용자 스토리 (User Stories)

### Epic 1: 생산 계획 및 지시
- **US-PR-001** (Must) — 생산팀장으로서, 주간 생산 계획을 등록하고 일일 작업 지시로 자동 분할되어 라인별로 배정되기를 원한다.
- **US-PR-002** (Must) — 생산팀장으로서, 라인별·작업자별 일일 생산 지시서를 발행하고 모바일 푸시로 즉시 전달되기를 원한다.
- **US-PR-003** (Should) — 생산팀장으로서, ERP의 수주 정보를 자동 가져와 생산 계획에 반영하고 싶다.
- **US-PR-004** (Should) — 생산팀장으로서, 설비 가용률·작업자 배치 제약을 고려한 AI 자동 스케줄링 추천을 받고 싶다.
- **US-PR-005** (Nice) — 생산팀장으로서, 시뮬레이션을 통해 계획 변경 시 납기 영향을 사전 확인하고 싶다.

### Epic 2: LOT 트레이서빌리티
- **US-LOT-001** (Must) — 작업자로서, 원자재 LOT 바코드를 스캔하면 자동으로 작업지시서에 연결되기를 원한다.
- **US-LOT-002** (Must) — 작업자로서, 공정 진행 시 부모-자식 LOT 관계가 자동 기록되기를 원한다.
- **US-LOT-003** (Must) — 품질팀 담당자로서, 완제품 LOT를 입력하면 사용된 원자재 LOT, 공정 이력, 작업자, 설비, 품질 데이터를 한 화면에서 보고 싶다.
- **US-LOT-004** (Must) — 품질팀 담당자로서, 특정 원자재 LOT에 문제가 발생하면 영향받은 모든 완제품 LOT 목록을 1분 이내에 받고 싶다.
- **US-LOT-005** (Should) — 영업/CS팀으로서, 고객 출하 LOT 번호로 전체 생산 이력을 PDF 보고서로 자동 생성하고 싶다.
- **US-LOT-006** (Should) — 작업자로서, RFID/QR 코드로 LOT 정보를 스캔만으로 자동 인식되기를 원한다.

### Epic 3: BOM 관리
- **US-BOM-001** (Must) — 자재팀장으로서, 제품별 다단계 BOM(원자재→반제품→완제품)을 트리 구조로 등록·수정하고 싶다.
- **US-BOM-002** (Must) — 자재팀장으로서, BOM 버전 관리(유효 시작일/종료일)와 변경 이력을 추적하고 싶다.
- **US-BOM-003** (Must) — 자재팀장으로서, BOM 변경 시 진행 중인 생산 지시에는 영향을 주지 않고 신규 지시부터 적용되기를 원한다.
- **US-BOM-004** (Should) — 자재팀장으로서, 생산 지시 발행 시 BOM 기반 자재 소요량이 자동 계산되어 자재 차감 예약이 되기를 원한다.
- **US-BOM-005** (Nice) — 자재팀장으로서, 대체 자재(Alternative Material) 등록 및 우선순위 관리를 하고 싶다.

### Epic 4: 공정별 실적 자동 수집
- **US-PRC-001** (Must) — 시스템으로서, PLC/IoT 센서에서 공정별 생산 수량·불량 수량·가동 시간을 실시간 수집하여 MES에 기록한다.
- **US-PRC-002** (Must) — 작업자로서, 공정 시작/종료를 NFC 태깅 또는 버튼 한 번으로 처리하고 싶다.
- **US-PRC-003** (Should) — 생산팀장으로서, 공정별 사이클 타임·수율을 실시간 모니터링하고 표준 대비 이상 알림을 받고 싶다.
- **US-PRC-004** (Should) — 시스템으로서, 정전·통신장애 등 비상 시에도 로컬 캐싱 후 복구 시 자동 동기화한다.

- **확장성**: 라인 5개 → 20개 확장 시 무중단

- **동시성**: 동시 사용자 100명 이상 지원

## 7. 의존성 및 가정

### 의존성
- **ERP 시스템**: 수주 데이터 연동 (REST API 또는 DB 직접 조회)
- **PLC 인프라**: OPC-UA Gateway 구축 (인프라팀 협조 필요)
- **바코드/RFID**: Zebra 또는 Honeywell 단말기 도입
- **모바일**: 안드로이드 태블릿 라인별 1대 배치

### 가정
- 생산 라인 5개, 작업자 50명, 일일 LOT 발행 약 200건
- 식품 제조업 특성상 24시간 가동 가능성 (3교대)
- 식품 위생법상 LOT 추적 데이터 5년 이상 보관 필요
